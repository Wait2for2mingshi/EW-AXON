# 零部件寿命 AI 监控规划（首期可落地版）

## 目标
- 面向自动化设备零部件寿命预警场景，先建设一套可持续采集、可稳定落地、可逐步扩展的数据方案。
- 首期重点不是直接做“精准寿命预测”，而是先做“退化趋势识别 + 提前预警”。
- 首期原则：`不新增专业采集线`、`不依赖高频模拟量`、`优先复用现有 PLC / IO / 报警 / 产能 / 维修记录`。

## 首期实现原则
1. 先做容易接入的数据，不做必须额外加传感器、加采集卡、接驱动专用总线的数据。
2. 先做事件类、计数类、耗时类、结果类数据，不做高频波形类数据。
3. 先覆盖停机影响大、故障频率高、已有信号基础的部件。
4. 先用规则和基线做预警，后续数据积累够了再升级为模型预测。

## 当前首期最容易接入的数据源
- 产能/良率数据：来自现有产能 CSV。
- 报警数据：来自现有报警 CSV。
- IO 状态与动作结果：来自 PLC 现有输出点、输入点、到位信号、真空开关信号。
- 维护更换记录：来自人工登记、保养记录、备件更换记录。
- 工艺结果数据：来自现有 OK/NG、工位失败次数、重试次数。

## 首期不建议纳入的数据
以下数据并非没有价值，而是实现成本高、接入复杂、维护代价大，不适合作为首期范围：

- 电机实时电流、扭矩、编码器精细误差。
- 轴承振动、轴承温度、减速机振动频谱。
- 气缸压力曲线、流量曲线、泄漏量定量值。
- 真空模拟量曲线、真空漏率、气路流量。
- 夹爪夹持力、治具磨损量、刀具磨耗量在线模拟值。
- 传感器光强、相机亮度衰减曲线、专业诊断总线数据。

## 首期建议监控分类

### 1. 轴类/运动轴系
适用对象：
- 伺服轴
- 丝杆
- 导轨
- 联轴器
- 减速机
- 同步带轴组

首期建议采集参数：
- 轴动作次数
- 回零次数
- 到位超时次数
- 单次动作耗时
- 节拍变慢次数
- 轴相关报警次数
- 轴复位次数
- 连续失败次数

可选增强项：
- 驱动 ready / not ready 状态
- 驱动报警码
- 过载报警次数

首期不做：
- 电机电流
- 扭矩
- 振动
- 轴承温度

预警思路：
- 动作耗时持续变长
- 到位超时次数上升
- 回零频率异常增加
- 相同轴报警频率持续升高

### 2. 气缸/气动执行类
适用对象：
- 标准气缸
- 滑台气缸
- 夹爪气缸
- 挡停气缸
- 升降气缸

首期建议采集参数：
- 伸出次数
- 回退次数
- 伸出到位耗时
- 回退到位耗时
- 到位超时次数
- 连续重试次数
- 不到位报警次数
- 人工干预次数

建议数据来源：
- 电磁阀输出信号
- 气缸原位/动位感应器输入信号
- 报警记录

首期不做：
- 缸压曲线
- 泄漏量
- 流量曲线

预警思路：
- 伸出或回退耗时变长
- 不到位次数上升
- 某一方向故障集中出现
- 更换前连续重试次数显著增多

### 3. 真空吸附类
适用对象：
- 真空吸盘
- 真空发生器
- 真空电磁阀
- 真空开关
- 真空管路
- 过滤器

首期建议采集参数：
- 吸附次数
- 放料次数
- 真空到位耗时
- 真空到位超时次数
- 真空保持失败次数
- 掉料次数
- 重吸次数
- 吸附失败次数
- 相关 NG 次数

建议数据来源：
- 真空开关到位信号
- 真空输出控制信号
- 工艺结果 OK/NG
- 报警记录

首期不做：
- 真空模拟量绝对值
- 漏率
- 流量

预警思路：
- 真空建立时间持续变长
- 吸附失败率上升
- 掉料次数上升
- 重吸次数上升

### 4. 电磁阀/气动控制元件
适用对象：
- 单电控阀
- 双电控阀
- 阀岛
- 调压阀

首期建议采集参数：
- 阀动作次数
- 对应执行件不到位次数
- 动作后超时次数
- 相关报警次数
- 更换次数

建议数据来源：
- 电磁阀输出点
- 对应执行件到位信号
- 报警记录
- 维修更换记录

预警思路：
- 阀动作正常但执行件不到位次数增加
- 同一阀关联异常持续增加

### 5. 夹爪/治具/传动机构
适用对象：
- 机械夹爪
- 治具夹紧机构
- 定位销
- 同步带
- 链条
- 推料机构

首期建议采集参数：
- 夹紧次数
- 松开次数
- 夹紧到位耗时
- 夹紧失败次数
- 偏位 NG 次数
- 卡料次数
- 人工调整次数

建议数据来源：
- 夹紧输出信号
- 到位感应器
- 报警记录
- 工艺 NG 结果

首期不做：
- 张力值
- 磨损量
- 夹持力

预警思路：
- 夹紧耗时持续增加
- 偏位 NG 与卡料次数同步上升
- 人工调整频次明显增加

### 6. 工艺执行头/耗材类
适用对象：
- 刀具
- 批头
- 焊头
- 喷嘴
- 吸嘴
- 点胶阀

首期建议采集参数：
- 使用次数
- 工作小时
- 相关 NG 次数
- 重工次数
- 自动补偿次数
- 更换周期
- 更换前异常次数

建议数据来源：
- 产能/良率统计
- 工艺失败记录
- 人工更换记录

首期不做：
- 磨损模拟量
- 温度曲线
- 扭矩曲线

预警思路：
- 随使用次数增加，相关 NG 率同步上升
- 补偿次数上升
- 更换前短周期内异常连续增多

### 7. 传感器与检测反馈类
适用对象：
- 光电传感器
- 接近开关
- 压力开关
- 真空开关
- 编码器反馈到位信号

首期建议采集参数：
- 触发次数
- 未触发次数
- 触发超时次数
- 误触发报警次数
- 信号丢失次数
- 校准次数
- 更换记录

建议数据来源：
- 输入点信号
- 报警记录
- 维修记录

首期不做：
- 光强值
- 模拟量漂移曲线

预警思路：
- 触发失败率上升
- 同一传感器相关报警反复出现

## 首期推荐优先级

### P1：优先落地
- 气缸/气动执行类
- 真空吸附类
- 电磁阀/气动控制元件
- 夹爪/治具/传动机构

原因：
- 普遍已有 IO 点和到位信号。
- 动作与异常关系清晰，容易形成预警规则。
- 不需要额外专业采集硬件。

### P2：第二阶段接入
- 轴类/运动轴系
- 工艺执行头/耗材类

原因：
- 可以先做“弱监控”，以报警、超时、节拍、结果退化为主。
- 若未来具备驱动状态读取能力，再增强。

### P3：后续扩展
- 传感器与检测反馈类
- 高成本模拟量监测项

原因：
- 价值存在，但首期收益不一定高于实施成本。

## 首期统一指标类型
无论哪类部件，首期都建议统一落到以下几种指标，便于后续复用统一分析逻辑：

- 计数类：动作次数、循环次数、触发次数、使用次数。
- 耗时类：指令到到位耗时、建立时间、回零时间、节拍时间。
- 异常类：超时次数、报警次数、重试次数、复位次数、人工干预次数。
- 结果类：成功率、失败率、掉料率、偏位率、相关 NG 率。
- 维护类：保养时间、更换时间、更换前征兆、更换后恢复情况。

## 预警逻辑建议
首期建议先用规则和基线，不要一上来就做复杂模型：

- 绝对阈值规则：例如到位超时次数超过阈值。
- 趋势规则：例如最近 7 天平均动作耗时持续升高。
- 比例规则：例如吸附失败率、夹紧失败率超过阈值。
- 更换前征兆规则：例如更换前 3 天异常次数显著高于平时。
- 组合规则：例如“动作耗时变长 + 相关报警增加 + NG 上升”同时满足时触发高等级预警。

## 结论
- 首期应聚焦 `气缸`、`真空吸附`、`电磁阀`、`夹爪/治具` 四类部件。
- 首期应优先采 `动作次数`、`动作耗时`、`到位超时`、`失败次数`、`报警次数`、`更换记录` 这类易实现数据。
- `轴类` 可以纳入首期，但只建议做弱监控，不建议首期追求驱动级诊断。
- 高成本、高接入复杂度的数据统一放到后续增强阶段，避免拖慢整体落地节奏。
